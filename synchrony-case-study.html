Predicting Customer Spending Patterns with LSTM Neural Networks

Building a deep learning forecasting system to predict $B-scale customer spending across 14K+ accounts using historical transaction data, time-series analysis, and adaptive LSTM architectures.

Challenge

The Problem

Synchrony Financial needed accurate predictions of customer spending patterns to optimize credit risk management, identify growth opportunities, and allocate resources efficiently. Traditional statistical models (ARIMA, exponential smoothing) struggled with non-linear customer behavior patterns, seasonal variations, and account-level heterogeneity.

The core challenge: predict future spending across diverse customer segments with minimal false negatives, enabling proactive risk mitigation and targeted customer engagement. Each account has unique spending patterns shaped by income cycles, purchasing behavior, life events, and macroeconomic factors.

Data Pipeline

Starting with raw transaction data, I built a comprehensive analytical foundation:

  • → Data Aggregation: Parsed transaction timestamps, aggregated spending by account and month
  • → Pivot Creation: Built monthly spending matrices (rows = accounts, columns = months)
  • → Handling Missing Data: Forward-filled sparse months, replaced NaN with account-level zeros
  • → Feature Normalization: Applied per-account MinMax scaling to preserve relative spending patterns

Key Design Decisions

Why This Architecture?
  • Account-Level Normalization: Each account normalized by its own min/max values to preserve relative patterns while handling extreme outliers
  • 8-Month Input Window: Captures 2 full quarters of historical data—balancing recency with seasonal variation
  • Per-Instance Models: Separate LSTM models for Q4 (Oct-Dec) and Q3 (Apr-Sep) predictions—seasonal models outperform generic ones
  • Recursive Forecasting: Multi-step ahead prediction by feeding previous outputs into next predictions for 6-month horizon
  • Early Stopping: Prevents overfitting by monitoring validation loss with 5-epoch patience

Learnings

Technical & Business Insights

Challenge: Extreme Outliers in Individual Accounts

Some accounts showed 10x spending spikes during holidays (Black Friday, Christmas) or drops due to seasonal income variations. Initial global normalization failed—the model predicted everything as near-average.

Solution: Switched to per-account normalization. Each account's sequence is normalized by its own min/max values before being fed to the LSTM. This preserves the relative pattern shape while handling extreme outliers. Trade-off: predictions are denormalized using account-specific scaling factors, adding one extra step in the pipeline.

Challenge: Seasonal vs. Multi-Year Trends

An 8-month window captures about 2 quarters, but spending patterns often have multi-year cycles (employment changes, life events). LSTM alone sometimes missed longer-term context.

Solution: Built separate seasonal models—one for Q4 (Oct–Dec, includes holidays) and another for Q3 (Apr–Sep, less volatile). This allows the model to learn season-specific patterns with higher granularity. When applied to new data, we route predictions through the appropriate seasonal model.

Challenge: Multi-Step Predictions Drift

When predicting 6 months ahead, errors compound. Predicting month 6 requires the model to use its own predictions from months 1–5 as input, which amplifies earlier mistakes.

Solution: Implemented recursive forecasting with error tracking. At each step, we not only predict the next month's spending but also estimate prediction uncertainty (confidence bands). High-variance predictions are flagged for manual review. Additionally, we ensemble predictions from multiple model checkpoints to reduce variance.

Business Insight: Customer Segmentation Emerges Naturally

By analyzing prediction errors across accounts, we discovered distinct customer personas:

Segment A (37%): Highly predictable, consistent monthly spenders | MAE = $1,200
Segment B (48%): Moderate volatility, seasonal patterns | MAE = $2,800
Segment C (15%): High volatility, erratic spending | MAE = $5,100

This segmentation can drive targeted interventions—reliable payers can receive lower-friction credit offers, volatile segments need additional review protocols.

Next Steps

Opportunities for Expansion

  • Incorporate External Features: Add macroeconomic indicators (unemployment, inflation, credit card approval rates) as LSTM input features—could capture 5–10% additional variance
  • Attention Mechanisms: Implement transformer-based architecture with attention weights to isolate which historical months most influence future spending
  • Explainability: Use SHAP or LIME to decompose predictions per account—why did we predict $X? (e.g., "similar to 2023 May spending, +15% adjustment")
  • Real-Time Updates: Deploy as live API with online learning—retrain monthly on latest data, adapt to new customer behaviors
  • Uncertainty Quantification: Quantile regression LSTM to output prediction intervals (10th, 50th, 90th percentiles) instead of point estimates

Key Takeaways

This project demonstrated that LSTMs can capture customer spending patterns at scale when properly normalized and segmented. The main success factors were:

1. Domain-Aware Normalization
Account-level normalization preserved patterns while handling outliers better than global scaling

2. Seasonal Decomposition
Separate models for different seasonal contexts reduced prediction error by ~12%

3. Uncertainty Tracking
Monitoring prediction variance allowed us to focus manual review on high-risk forecasts

4. Explainability Over Raw Accuracy
A 0.76 R² model with clear error patterns is more actionable than a 0.82 R² black box

The model is now deployed in production, generating monthly forecasts for 14K+ accounts and informing credit decisions, collection strategies, and customer segmentation across the portfolio.
